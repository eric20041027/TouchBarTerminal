# Touch Bar 番茄鐘 / 專注狀態列

> 由 idea-refine 產出（2026-06-19）。承接 [touchbar-direction.md](../intent/touchbar-direction.md)。
> 從 8 個點子中，使用者選定方向 4。

## Problem Statement

**HMW** 讓這台 M2 的 Touch Bar 顯示「會主動變化、瞄一眼就有用」的東西，
讓使用者**養成低頭看 Touch Bar 的習慣**——用一個會持續倒數的專注計時器，
利用「它一直在動」這個特性反過來把注意力拉過去。

## Recommended Direction

做一個 **Touch Bar 原生番茄鐘**：Touch Bar 上一顆「▶ 開始」按鈕，點下去開始
25 分鐘倒數，倒數直接呈現在 Touch Bar 那條光帶上（這是手機/選單列番茄鐘給不了的
視覺——**只有 Touch Bar 做得到**）。到點時整條閃爍/變色 + 發系統通知。

為什麼是這個而不是 Claude 橋接（點子 3）：3 的致命風險是 Claude 桌面版沒有公開
的「接收外部文字開新對話」API，一鍵丟進去得靠脆弱的自動化，違反「純公開 API、
穩定」的 constraint。番茄鐘純公開 API、穩、複雜度對 Swift 初學者友善，且**完美
命中主要目標 (a) 養成習慣**——因為它會動，使用者自然會瞄。

這是維他命不是止痛藥，使用者也認了——獨特性必須來自「只有 Touch Bar 的整條光帶
能這樣呈現倒數」，而不是番茄鐘本身（番茄鐘 app 滿街都是）。

## Key Assumptions to Validate

- [ ] **「會動的倒數」真的能養成看 Touch Bar 的習慣** —— 用幾天後問自己：有沒有
      自然開始瞄 Touch Bar？（這是整個方向的核心賭注，做完要誠實回測。）
- [ ] **Touch Bar 那條光帶呈現倒數，視覺上夠獨特、夠爽** —— 做出來看一眼就知道，
      若只是個數字跳動跟選單列番茄鐘沒差，獨特性就垮了 → 要在「整條進度光帶/漸層」
      下功夫。
- [ ] **到點提醒（閃爍+通知）抓得住注意力** —— 專注時可能根本沒看 Touch Bar，
      系統通知（有聲音）是保險。

## 習慣回測（排程 2026-06-23，做完 MVP 後 3 天）

MVP 於 2026-06-20 上線。3 天後用**真實行為**回測核心賭注，回測時誠實回答：

1. **行為：** 這幾天有沒有「**沒按番茄鐘時也自然瞄了 Touch Bar**」幾眼？
   （習慣 = 不為了用功能也會看。）
2. **使用：** 有沒有**真的按 ▶ 開始番茄鐘**？幾次？還是一次都沒按？
3. **判決：**
   - 有看 + 有按 → 賭注成立，方向對，可接著做下一個「會動的資訊」。
   - 幾乎沒看也沒按 → **賭注輸了**。代表問題不在「放什麼」，而在 Touch Bar
     這個位置搶不到注意力 → **回頭重想方向，不是加功能**（見 touchbar-direction.md）。

## MVP Scope

**做：**
- Touch Bar 一顆「▶ 開始」按鈕（與現有 git 模式並存，或獨立一區）。
- 點擊 → 開始 25 分鐘倒數，Touch Bar 顯示剩餘時間 + 一條會縮短的進度光帶。
- 倒數中按鈕變「⏸ 停止/重置」。
- 最後階段（如剩 1 分）光帶變紅；到 0 → 整條閃爍 + 發 macOS 系統通知（含聲音）。
- 純邏輯（倒數狀態機、剩餘時間格式化、進度比例）先寫測試（TDD，見 CLAUDE.md）。

**不做（見下）。**

## Not Doing (and Why)

- **全域熱鍵啟動** —— MVP 用 Touch Bar 按鈕開始就好；熱鍵之後要再加一個
  GlobalHotKey，先驗證核心價值再說。
- **自動偵測專不專心 / 被動計時** —— 「自動偵測」很難、不可靠，超出 MVP。
- **今日累計專注時間 / 統計** —— 等核心倒數驗證有用了再加。
- **自訂時長 / 長短休息循環** —— 先固定 25 分鐘；標準番茄鐘的完整循環之後再說。
- **Claude 橋接（點子 3）** —— 風險高（無公開 API），押後。

## Open Questions

- 番茄鐘按鈕要跟 git 模式**共用 ⎇ 那套切換**，還是獨立常駐一區？（現在 Touch Bar
  已有：左路徑/輸入 + ⎇ + 輸出/git。番茄鐘塞哪、會不會又擠版面，要設計。）
- 倒數中如果使用者切走 app（Touch Bar 換成別的 app 的），計時器要繼續跑（背景），
  回來時顯示正確剩餘時間——這點要確認現有架構（背景常駐）支援。
